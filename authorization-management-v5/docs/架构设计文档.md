# 架构设计文档：授权书管理系统 V5

## 一、技术选型

### 1.1 前端技术栈
- 框架：Vue 2.x（原生，不使用Element UI）
- 路由：Vue Router
- HTTP客户端：axios
- 构建工具：Vue CLI

### 1.2 后端技术栈
- 框架：Spring Boot 2.7.18
- JAX-RS实现：Jersey
- ORM：MyBatis
- 数据库：PostgreSQL
- 数据库迁移：Flyway
- 测试框架：JUnit 5 + Spring Boot Test

## 三、规则配置JSON结构设计

### 3.1 条件组结构
```json
{
  "logic": "AND",
  "conditions": [
    {
      "type": "condition",
      "fieldId": 1,
      "fieldName": "金额",
      "operator": ">=",
      "compareType": "NUMBER",
      "compareValue": "100",
      "unit": "万元"
    },
    {
      "type": "group",
      "logic": "OR",
      "conditions": [
        {
          "type": "condition",
          "fieldId": 2,
          "fieldName": "区域",
          "operator": "=",
          "compareType": "TEXT",
          "compareValue": "华东"
        }
      ]
    }
  ]
}
```

### 3.2 字段说明
- logic: 条件间关系（AND/OR）
- conditions: 条件数组
- type: condition-条件项, group-子条件组
- fieldId: 规则参数ID
- fieldName: 规则参数名称
- operator: 运算符（>, <, =, >=, <=, !=）
- compareType: 对比类型（FIELD-对比字段, NUMBER-数值, TEXT-文本）
- compareValue: 对比值
- unit: 单位（数值类型时使用）
- compareFieldId: 对比字段ID（对比类型为FIELD时使用）

## 六、状态流转

```
草稿(DRAFT) ─发布→ 已发布(PUBLISHED) ─失效→ 已失效(INVALID)
    ↓                    ↓                    ↓
  可删除              可删除               可删除
                       可失效
```
